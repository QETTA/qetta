# QETTA 실행 계획 v2.0

## Executive Summary

이 문서는 QETTA의 **제품 완성 (90%) → 비즈니스 완성 (10% → 100%)**을 위한 실행 계획입니다. 2026년 시장 환경을 고려할 때, 핵심 전략은 "투자 유치 전 매출 증명"입니다.

### 핵심 원칙

2026년 한국 스타트업 시장의 현실을 반영합니다. 투자 빙하기가 지속되고 있으며, 아이디어보다 실증 데이터와 고객 레퍼런스를 요구합니다. 과거에는 투자 유치가 성공의 지표였지만, 이제는 실제 고객 확보와 매출 달성이 더 중요한 지표입니다.

따라서 QETTA의 전략은 다음과 같습니다. 첫째, 투자를 구하지 않습니다. 둘째, 고객을 먼저 확보합니다. 셋째, 매출이 투자를 부릅니다.

### 성공 지표

| 시점 | 목표 | 달성 시 의미 |
|-----|-----|------------|
| 3개월 | 유료 고객 10곳 + 월 매출 500만원 | PMF 초기 신호 |
| 6개월 | 유료 고객 30곳 + 월 매출 1,500만원 | PMF 검증 완료 |
| 12개월 | 유료 고객 62곳 + 월 매출 1,200만원 | 투자 유치 가능 상태 |

## Phase 1: First Revenue (0-3개월)

### 목표

첫 번째 유료 고객을 확보하고, 제품-시장 적합성(PMF)의 초기 신호를 확인합니다.

### 핵심 지표

| 지표 | 목표 | 측정 방법 |
|-----|-----|----------|
| Pilot 고객 | 5곳 | 무료 3개월 제공 |
| 유료 전환 | 3곳 | Pilot → PRO 전환 |
| 월 매출 | 300만원+ | PRO × 3곳 |
| NPS | 40+ | 월별 설문 |

### Week 1-2: 인프라 구축

첫 주에는 비즈니스 모델을 제품에 반영합니다.

**가격 페이지 업데이트**가 최우선입니다. 현재 "무료 30일 + 엔터프라이즈 문의" 구조를 "FREE + PRO + TEAM + ENTERPRISE" 4티어 구조로 변경합니다. 연간 결제 할인(2개월 무료)을 표시하고, EXPORT 크레딧 섹션을 추가합니다.

```typescript
// 가격 페이지 구조
const PRICING_PAGE = {
  tiers: [
    { name: "FREE", price: 0, cta: "시작하기" },
    { name: "PRO", price: 99000, cta: "시작하기", badge: "POPULAR" },
    { name: "TEAM", price: 290000, cta: "시작하기" },
    { name: "ENTERPRISE", price: "문의", cta: "영업팀 문의" }
  ],
  toggle: ["월간 결제", "연간 결제 - 2개월 무료"],
  exportCredits: {
    bundles: [10, 50, 100, 300],
    prices: [100000, 400000, 700000, 1800000]
  }
};
```

**결제 시스템 연동**도 이 시기에 완료합니다. Stripe 또는 토스페이먼츠를 연동하고, 구독 결제 흐름(월간/연간)을 구현합니다. 크레딧 구매 흐름도 구현합니다.

**대시보드 핵심 페이지 추가**는 `/dashboard/blocks` 페이지를 최우선으로 합니다. BLOCK이 QETTA의 핵심 가치인데 현재 이를 보여주는 UI가 없습니다. "내 BLOCK" 카드와 "마켓플레이스" 섹션을 추가합니다.

### Week 3-4: Pilot 고객 확보

**고객 발굴 채널**을 설정합니다.

첫 번째 채널은 창업자 네트워크입니다. 스타트업 커뮤니티(디캠프, 마루180, 팁스타운)에서 정부지원사업 신청 예정인 스타트업을 발굴합니다. "AI 바우처 신청 예정이신 분 계신가요? 무료로 3개월 써보실 분 모십니다"라는 메시지로 접근합니다.

두 번째 채널은 정부지원사업 컨설팅 업체 제휴입니다. 이미 고객을 보유한 컨설팅 업체와 제휴하여 수수료 기반 협력 모델을 제안합니다.

세 번째 채널은 콘텐츠 마케팅입니다. "AI 바우처 신청 가이드", "스마트공장 사업계획서 작성법" 등의 콘텐츠를 블로그/유튜브에 게시합니다.

**Pilot 조건**은 다음과 같습니다. 기간은 3개월 무료이고, 대상은 정부지원사업 신청 예정 중소기업입니다. 대가로 주 1회 피드백 미팅, 성공 시 케이스 스터디 작성 동의, 유료 전환 우선 협의를 요청합니다.

### Week 5-8: 제품 개선 및 전환

**피드백 루프**를 구축합니다. 주 1회 30분 피드백 미팅을 진행하고, "무엇이 불편한가?", "무엇이 가장 유용한가?"를 질문합니다. 피드백을 바탕으로 2주 스프린트로 개선합니다.

**유료 전환 준비**를 합니다. Pilot 시작 2개월 차에 전환 논의를 시작합니다. "정식 출시 후에도 계속 쓰실 의향이 있으신가요?"라고 물어보고, 유료 전환 시 3개월 50% 할인 제공합니다.

## Phase 2: Revenue Proof (3-6개월)

### 목표

PMF를 검증하고, 반복 가능한 판매 모델을 구축합니다.

### 핵심 지표

| 지표 | 목표 | Phase 1 대비 |
|-----|-----|-------------|
| 유료 고객 | 30곳 | 10배 성장 |
| 월 매출 | 1,500만원 | 5배 성장 |
| NPS | 50+ | +10 |
| 월 이탈률 | 5% 이하 | - |

### AI 바우처 공급기업 등록

AI 바우처 공급기업으로 등록되면 정부가 고객을 연결해줍니다. 이것이 가장 효율적인 고객 획득 채널입니다.

**등록 요건**을 확인합니다. AI 솔루션 보유 기업이어야 하고, 기업 신용등급 B- 이상이 필요합니다. 최근 3년 이내 AI 관련 사업 실적이 있어야 하며, AI 전문인력 1인 이상이 필요합니다.

**등록 절차**는 다음과 같습니다. 먼저 NIPA 공급기업 등록 시스템에 접속합니다. 기업정보와 솔루션 정보를 입력하고, 심사를 거쳐 승인을 받습니다. 예상 소요 기간은 2-4주입니다.

**등록 후 효과**는 상당합니다. AI 바우처 수요기업이 QETTA를 선택할 수 있게 됩니다. 정부가 70-80% 비용을 지원하므로 고객 부담이 낮아집니다. 실질적으로 고객 획득 비용(CAC)이 0에 가까워집니다.

### 마케팅 채널 구축

**콘텐츠 마케팅**을 본격화합니다.

블로그 콘텐츠로는 "2026 AI 바우처 완벽 가이드", "스마트공장 구축 성공 사례 10선", "정부지원사업 사업계획서 작성 노하우" 등을 작성합니다. 주 2회 발행하고, SEO 최적화하여 장기적 유입을 만듭니다.

유튜브 콘텐츠로는 "QETTA로 사업계획서 30분 만에 작성하기" 실제 시연 영상, "AI 바우처 수혜 기업 인터뷰" 케이스 스터디 영상을 제작합니다. 월 2회 업로드합니다.

**세미나/웨비나**도 진행합니다. "2026 정부지원사업 활용 전략" 월 1회 온라인 세미나를 개최합니다. 참석자에게 QETTA 무료 체험 + 상담을 제공합니다.

### 팀 구성 (선택)

매출이 500만원을 넘으면 팀 확장을 검토합니다.

**필요 포지션**은 두 가지입니다. 첫째, 영업/CS 담당 1명입니다. Pilot 운영, 고객 온보딩, 유료 전환을 담당합니다. 둘째, 마케팅 담당 1명입니다. 콘텐츠 제작, 세미나 운영, 리드 관리를 담당합니다.

**채용 기준**은 다음과 같습니다. 정부지원사업 경험자를 우대합니다. 초기 스타트업 경험이 있으면 좋습니다. 자기 주도적으로 일할 수 있어야 합니다.

## Phase 3: Scale-Up (6-12개월)

### 목표

검증된 모델을 확장하고, 투자 유치 준비를 합니다.

### 핵심 지표

| 지표 | 6개월 | 12개월 | 비고 |
|-----|------|-------|-----|
| 유료 고객 | 30곳 | 62곳 | 2배 성장 |
| 월 매출 | 1,500만원 | 1,200만원 | 연간 1.4억 |
| ARR | - | 1.4억원 | 투자 가능 기준 |
| 팀 규모 | 2-3명 | 5-7명 | 필요 시 |

### 스마트공장 도메인 확장

AI 바우처에서 PMF가 검증되면 스마트공장으로 확장합니다.

**확장 근거**는 다음과 같습니다. 2026년 스마트공장 예산이 1,695억원입니다. 제조업 BLOCK(SEMICONDUCTOR, MACHINERY, ELECTRONICS, MANUFACTURING)을 이미 보유하고 있습니다. AI 바우처 고객 중 제조업 비중이 높습니다.

**확장 전략**은 AI 바우처 고객 업셀링입니다. "AI 바우처로 시작하셨으니, 스마트공장도 신청해보시겠습니까?"라고 제안합니다. 같은 BLOCK 재사용으로 추가 개발 없이 확장이 가능합니다.

### EXPORT 크레딧 활성화

글로벌 입찰(SAM.gov, UNGM) 기능을 활성화합니다.

**타겟 고객**은 해외 진출 희망 기업입니다. 기존 유료 고객 중 수출 실적이 있거나 계획 중인 기업을 파악합니다. "해외 입찰도 지원한다"는 것을 알립니다.

**매출 기여 예상**은 다음과 같습니다. PRO 고객 20%가 크레딧 구매 시 1,200만원/년, TEAM 고객 50%가 크레딧 구매 시 1,500만원/년입니다.

### 투자 유치 준비

월 매출 1,000만원 이상 + 성장률 유지 시 투자 유치를 검토합니다.

**투자 유치 타이밍**은 다음 조건이 충족될 때입니다. MRR 1,000만원 이상이어야 합니다. MoM 성장률 10% 이상이어야 합니다. NPS 50 이상이어야 합니다. 이탈률 5% 이하여야 합니다.

**투자 유치 전략**은 다음과 같습니다. Pre-A 라운드로 5-10억원을 목표로 합니다. 타겟 투자자는 B2B SaaS 전문 VC와 정부지원사업 이해도 있는 VC입니다. 피치 포인트는 "도메인 특화 AI SaaS", "정부 예산 연동 시장", "매출 증명된 PMF"입니다.

## 리스크 관리

### 리스크 1: 고객 획득 실패

**리스크**: Pilot 고객을 못 모음
**확률**: 30%
**영향**: 높음

**완화 전략**:
1. AI 바우처 공급기업 등록 (정부가 고객 연결)
2. 정부지원사업 컨설팅 업체 제휴 (이미 고객 있는 채널)
3. 무료 티어 오픈 (진입 장벽 제거)

**대응책**: B2B 영업 인력 채용 또는 외주

### 리스크 2: 경쟁사 진입

**리스크**: ChatGPT/Notion AI가 유사 기능 출시
**확률**: 50%
**영향**: 중간

**완화 전략**:
1. 도메인 특화 BLOCK은 범용 AI가 따라오기 어려움
2. Company BLOCK의 학습 데이터가 해자(Moat)
3. 정부지원사업 규정 전문성은 빠르게 복제 불가

**대응책**: EXPORT 차별화로 니치 확보

### 리스크 3: 기술 부채

**리스크**: 빠른 성장 시 시스템 불안정
**확률**: 40%
**영향**: 중간

**완화 전략**:
1. 이미 Next.js 15 + 모던 스택으로 구축됨
2. SSOT 패턴으로 데이터 일관성 확보
3. 테스트 코드 (Playwright, Vitest) 이미 존재

**대응책**: 시니어 개발자 채용

### 리스크 4: 자금 소진

**리스크**: 투자 전 자금 소진
**확률**: 30%
**영향**: 치명적

**완화 전략**:
1. PRO 티어 조기 매출화
2. 정부지원사업(TIPS, 창업성장) 자금 활용
3. EXPORT 크레딧으로 현금흐름 확보

**대응책**: 외주 개발 프로젝트 수주 또는 비용 절감
