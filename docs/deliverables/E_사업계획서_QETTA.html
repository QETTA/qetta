2. 시장 문제 (Problem)

2.1 “신청”은 도구가 있지만, “정산~감사”는 수작업입니다

e나라도움은 보조금 신청과 집행 등록을 디지털화했지만, 정산 증빙 검수·패키징·감사 대비는 여전히 수작업입니다. 회계법인은 매 건마다 부처별로 다른 서식과 규칙을 수동으로 확인하고, 반려 시 처음부터 재작업합니다.

2.2 4대 Pain Point

#문제수치 근거결과
P1 반려 반복 → 건당 적자 반려 재작업 인건비 80~150만원 × 평균 2.3회 건당 184~345만원 손실. 감사보수 1,520만원 대비 적자 구간 진입.
P2 속인화 → 퇴사 시 붕괴 CPA 합격자 수습등록률 26% (2025) 정산 노하우가 담당자 머릿속에만 존재. 채용도 어려움.
P3 감사/점검 확대 → 본업 올스톱 환수 집행 3년 새 3배 증가 소급 점검으로 3~5년 전 건도 대응 필요. 감사·세무 업무 밀림.
P4 환수 1건 = 평판 파괴 공공재정환수법 5배 제재부가금. 3,000만원 이상 명단 공표. 해당 매출 라인 소멸. 법인 신뢰 회복 불가.

2.3 Why Now? (2026 타이밍)

  1. 공공재정환수법 본격 시행 (2024~) → 5배 제재부가금 → 법인 리스크 인식 급증
  2. 부정수급 적발 역대 최대 (2024: 630건 / 493억원) → 감시 강화 추세 확정
  3. 감사보수 3년 연속 하락 + 인력난 → 비용 절감 자동화 도구 수요 급증
  4. 대형 법인 AI 도입 시작 → 소형/중견은 도구 부재 → “뒤처지면 안 된다” 인식 확산
  5. e나라도움 정산 기능 부재 → 시스템 공백 그대로. 3년 내 해소 전망 없음.

3. 솔루션 (Solution)

3.1 제품 포지셔닝

e나라도움을 대체하지 않습니다. 제출 직전에 품질을 잡아주는 검수 레이어입니다.

3.2 3단계 QA 프로세스

A. 품질검수

증빙 누락·부적격·비목 불일치 자동 감지. 부처별 서식 적합성 검증.

→

B. 노하우 구조화

부처별 정산 룰 + 법인 자체 체크리스트 시스템 내장. 담당자 교체에도 품질 유지.

→

C. 감사 대비 패키징

4종 산출물을 하나의 패키지로 생성. 점검 당일 즉시 대응.

3.3 1건 = 4종 산출물 패키지

#산출물내용
1정산서집행내역 기반 정산서 자동 구성
2증빙목록비목별 증빙 매칭 + 누락 항목 식별
3성과보고 초안KPI 기반 성과보고서 초안 생성
4감사 체크리스트주요 지적사항 사전 점검 + 근거 규정 링크 + 검토 로그

• 4종이 모두 생성되고, 담당 전문가(CPA)가 승인해야 1건으로 과금됩니다.

3.4 C-0: PII 자동 보호 (구매 차단 요인 해소)

회계법인의 #1 도입 장벽 = “고객 개인정보가 외부로 나가면 안 된다”

  • 증빙 내 주민번호·계좌번호·연락처를 업로드 즉시 자동 탐지 → 마스킹
  • 마스킹 완료 데이터만 처리. 원본은 국내 서버에 AES-256 암호화 분리 저장
  • 처리 로그 실시간 제공. 개인정보보호법(PIPA) 준수.
  • 이 기능이 없으면 파일럿 자체가 불가 → P0 (절대 필수)

7. 경쟁 환경

7.1 경쟁 지도

구분플레이어영역QETTA 대비
직접 경쟁 없음 (0개) 정산 검수 특화 SaaS 국내 시장 공백. 진입 장벽 = 부처별 정산 규칙 DB + PII 처리.
간접 경쟁 정산 대행 업체 회계법인/수행기관 외주 건당 50~100만 vs QETTA 18~25만. 비용 1/3 수준.
간접 경쟁 내부 수작업 엑셀 + 담당자 경험 속인화 리스크. 반려 반복. QETTA로 구조화.
인접 e나라도움 보조금 신청/집행 등록 정산 검수 기능 없음. QETTA = e나라도움 앞단 QA.
인접 세금신고 SaaS (삼쩜삼 등) 개인/소기업 세무 보조금 정산과 무관. 카테고리 자체가 다름.
인접 ERP (SAP, 더존 등) 회계/자원관리 보조금 정산 특화 모듈 없음. 범용 도구.

7.2 방어 가능성 (Moat)

  1. 부처별 정산 규칙 DB: 축적할수록 정확도 향상. 신규 진입자 동일 DB 구축에 6개월+.
  2. 법인별 커스텀 룰: 법인이 자체 체크리스트를 입력할수록 전환 비용(switching cost) 상승.
  3. 리퍼럴 네트워크: Pro 법인이 고객사를 소개할수록 네트워크 효과 발생. 자발적 확산 구조.
  4. 감사 이력 아카이브: 사용 기간이 길어질수록 감사 대비 데이터 축적 → 이탈 비용 증가.
  5. PII 처리 신뢰: 개인정보 처리 실적 + 로그 투명성 = 회계법인 신뢰 자산. 후발 주자 대비 선점 우위.

법인이 무료 체험 후 유료 전환하지 않음		 중 높음 • ROI 리포트 자동 생성 (반려 방지 건수 × 비용 산출)
• 12개월 가격 고정 인센티브
• 파일럿 중 밀착 지원 (대표 직접)
R2 PII 마스킹 누수
개인정보 미탐지 → PIPA 위반		 중 높음 • 정규식 + 패턴 이중 탐지
• 마스킹 실패 시 전송 차단 (fail-safe)
• PII 마스킹 실패 = PagerDuty P0 즉시 알림
R3 부처 서식 변경
정산 규칙/서식이 예고 없이 변경		 높음 중 • 규칙 DB 분리 (코드와 독립적 업데이트)
• 서식 변경 모니터링 + 법인 피드백 채널
• 변경 감지 시 24시간 내 규칙 업데이트
R4 Claude API 의존
API 장애/가격 인상/서비스 중단		 낮음 중 • 핵심 검수는 규칙 엔진 (Claude 불필요)
• Claude 장애 시 초안 생성만 대기 (검수 자체는 정상)
• 대체 LLM (GPT-4, Gemini) 프롬프트 사전 준비
R5 경쟁사 진입
대형 회계 SaaS 또는 e나라도움 자체 기능 추가		 낮음 중 • 선점 + 법인별 커스텀 룰 축적 = 전환 비용

• 리퍼럴 네트워크 효과로 자발적 확산 구조 구축

• PII 처리 로그 실시간 제공으로 투명성 확보
• 보안 감사 보고서 연 1회 제공 (Firm)

Kill Criteria (사업 중단 기준)

  • 파일럿 10개 법인 진행 후 유료 전환 2개 미만 (<20%) → 제품-시장 적합성 재검토
  • Y1 종료 시점 월 매출 200만원 미만 → 피봇 또는 중단 결정
  • PII 사고 발생 시 → 즉시 서비스 중단 → 원인 해소 후 재개

핵심 투자 포인트 요약

1. 시장 공백: 정산 특화 SaaS 국내 0개. 225,710 보조사업, 254 회계법인 대상 블루오션.
2. 규제 순풍: 공공재정환수법 5배 제재 + 부정수급 적발 역대 최대 → 수요 급증 타이밍.
3. 극한 마진: 건당 원가 720원 / 매출 18~25만원 → 마진 99.6%+. 소프트웨어 한계비용 ~0.
4. 자체 확산: 리퍼럴 파트너 모델. 법인이 고객에게 소개 → 네트워크 효과 → 낮은 CAC.
5. 부트스트랩 가능: Y1 필요 자금 500만원. 매출 = 거의 순이익 구조 → 외부 의존도 최소.
